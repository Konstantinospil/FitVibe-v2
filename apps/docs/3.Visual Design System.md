# FitVibe Visual Design Guide (VDG) · Option A · v1.0

**Summary.** This guide is the single source of truth for FitVibe’s look & feel. It locks the **Option A color/vibe canon** (your original seeds), defines tokens (Tailwind-ready), and specifies patterns for key screens with WCAG 2.1 AA baked in.

---

## 1) Principles

- **Energy × Discipline × Balance.** UI reflects training intensity and control.
- **Minimal & professional.** Clarity over ornament; strong hierarchy, generous spacing.
- **Accessible by default.** Keyboard-first, screen-reader friendly, color-blind safe.

## 4) Typography

- **Families:** Roboto Flex (variable) / Inter; Monospace: IBM Plex Mono.
- **Scale**
  - H1 3.0rem / 600 / 1.2
  - H2 2.0rem / 600 / 1.3
  - H3 1.5rem / 500 / 1.35
  - Body 1.125rem / 400 / 1.6
  - Caption 0.875rem / 400 italic / 1.4
  - Button 1.0rem / 600 uppercase / 1.2
- **Rules:** 65–75ch line length; supports 200% text scaling; avoid ALL-CAPS in paragraphs; ensure ≥4.5:1 contrast.

**CSS starter**


- Use `@tailwindcss/forms`; clear labels, help text, and ARIA-live errors.

## 8) Key Screen Patterns

1. **Login / Registration**
   - Minimal fields, clear validation; “Login” / “Register” use Primary button.
   - Screen-reader friendly: labels, `aria-invalid`, live regions.
   - Keyboard order logical; escape closes modal dialogs.

2. **Dashboard**
   - Card grid; **Completed = green** markers; **Planned = indigo→purple gradient**.
   - Typography scale respected; responsive 12-col grid; high contrast.

3. **Planner**
   - Drag & drop with keyboard alternative; event tooltips show sets/reps/load/RPE.
   - Calendar has ARIA roles & roving tabindex; undo available.

4. **Logger**
   - Mobile-first stepper; big touch targets; inline completion %; validation announcements.

5. **Progress Dashboard**
   - Recharts with axis labels, units, tooltips, legend (not color-only).
   - Export CSV/JSON; provide data table toggle + textual summary.

6. **Feed / Community**
   - Infinite scroll or pagination; default privacy = **private**; public sessions visibly tagged; each card is its own ARIA region.

7. **Profile & Settings**
   - Tabs: Profile / Security / Privacy; avatar upload with **128×128** preview.
   - Destructive actions gated by confirm modal.

---

## 9) Accessibility Checklist (AA)

- [ ] All text/background pairs ≥ **4.5:1** (check light & dark).
- [ ] Focus styles: visible ring + non-color cue (glow/outline).
- [ ] Keyboard path: login → planner → logger → export (no traps).
- [ ] Form errors announced via ARIA; labels bound to inputs.
- [ ] Charts include data table toggle + textual summary.
- [ ] Pointer targets ≥44×44; spacing ≥8px between adjacent controls.
- [ ] `prefers-reduced-motion` respected.

---

## 10) Content & Microcopy

- Voice: concise, encouraging, non-judgmental.
- Labels: action-first (“Start Session”, “Save Set”).
- Errors: specific + remediation (“Weight must be ≥ 0 kg”).
- Privacy nudges: clarify when sharing publicly; default is private.

#### 8.4.7 Typography System

The typography for **FitVibe** conveys **clarity, precision, and professionalism**.
It avoids decorative styles in favor of a **neutral, functional aesthetic** that supports readability across training dashboards, mobile interfaces, and data-dense components.

##### 8.4.7.1 Design Principles

- **Modern & Neutral:** Typography is clean and understated - focused on legibility, not personality.
- **Consistency:** Heading weights and spacing follow a unified rhythm; the same styles apply across all devices.
- **Function over style:** Size and spacing create hierarchy rather than color or ornament.
- **Performance:** System-rendered fonts reduce load time and maintain crispness on all screens.
- **Accessibility:** Conforms to WCAG 2.1 AA for size and contrast; supports text scaling to 200 %.

---

##### 8.4.7.2 Typefaces

#### 8.4.8 Accessibility & Contrast

- All text-to-background contrast ratios ≥ 4.5 : 1 (WCAG 2.1 AA).
- Color-blind-safe palette verified for deuteranopia and protanopia.
- Hover and focus states rely on both **color** and **shadow/glow** cues.
- Maintain a **maximum line length of 65-75 characters** for body text.
- Avoid all-caps in large blocks of text.
- Preserve at least **1.4 em vertical spacing** between paragraphs.
- Headings and UI labels must maintain **≥ 4.5 : 1** color contrast ratio.
- Typography is validated for clarity on **Retina / 4K** and **OLED** displays.
